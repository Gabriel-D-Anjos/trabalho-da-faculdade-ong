Instruções de entrega e observações:

Arquivos incluídos:
- index.html (página inicial institucional)
- projetos.html (lista de projetos e indicadores)
- cadastro.html (formulário de cadastro com validação HTML5)
- assets/styles.css (estilos responsivos e acessíveis)
- assets/scripts.js (pequenas interações, máscaras e validação)

Boas práticas implementadas:
- HTML semântico e acessível (header, nav, main, footer, fieldset, legend).
- Validação nativa HTML5 (atributos required, pattern, type).
- Máscaras de entrada simples para CPF, telefone e CEP implementadas em JS.
- Imagens com loading="lazy" e atributos width/height.
- Meta tags para SEO e viewport responsivo.
- Estrutura de pastas organizada pronta para enviar ao GitHub.
- Verifique o validador W3C se desejar validação formal dos arquivos.

Para entregar:
1) Configure um repositório público no GitHub com todo o conteúdo desta pasta.
2) Envie o link público conforme solicitado pelo professor/avaliador.

Observação: os arquivos project1.jpg e project2.jpg são placeholders — substitua por imagens otimizadas em múltiplos formatos antes de enviar ao GitHub.
